This page explains how to get Alchemist up and running on your computer using Docker. It's the easiest way to start because we've already done the hard work of packaging the software with all the drivers it needs to run fast.

Alchemist is designed to run in a Docker container for the best experience. Our official image bundles a compatible version of FFmpeg and all necessary drivers for hardware acceleration.

## Docker Compose (Recommended)

The easiest way to get started is with Docker Compose. Create a `docker-compose.yml` file with the following content:

```yaml
version: '3.8'

services:
  alchemist:
    image: ghcr.io/alchemist-project/alchemist:latest
    container_name: alchemist
    ports:
      - "3000:3000"
    volumes:
      - ./config:/config
      - /path/to/your/media:/media
    environment:
      - PUID=1000
      - PGID=1000
      - TZ=UTC
    devices:
      - /dev/dri:/dev/dri # Required for Intel QSV and AMD VAAPI
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu] # Required for NVIDIA NVENC
    restart: unless-stopped
```

## Running the Container

Once you've created your `docker-compose.yml` file, start Alchemist with:

```bash
docker compose up -d
```

Alchemist will now be available at `http://localhost:3000`.

## Hardware Acceleration Setup

Depending on your GPU vendor, you may need additional steps to enable hardware acceleration:

### NVIDIA
Ensure you have the [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html) installed on your host machine. This is what lets Docker "see" your graphics card.

### Intel/AMD
Ensure your user has permissions to access `/dev/dri` on the host. On most Linux distributions, you can add your user to the `render` or `video` group:

```bash
sudo usermod -aG render $USER
sudo usermod -aG video $USER
```
